# GET /api/v1/locales

Returns the locales that are active for the group, along with the group's default locale.

This endpoint is public — no authentication is required. It is intended for use by clients that need to know the available languages before a user is signed in (e.g. a login page or a public-facing view).

## Authentication

**None** — this endpoint is publicly accessible.

## Request

```
GET /api/v1/locales
```

No headers, query parameters, or request body required.

## Response

**200 OK**

```typescript
type LocalesResponse = {
  default: string    // The group's default locale (e.g. "en")
  active: string[]   // All active locale codes, including the default
}
```

Locale codes follow the [IETF language tag](https://en.wikipedia.org/wiki/IETF_language_tag) format. The platform supports: `en`, `es`, `it`, `ca`, `fr`, `de`.

Example:

```json
{
  "default": "en",
  "active": ["en", "es", "ca"]
}
```

## Errors

| HTTP | `error` | Cause |
|---|---|---|
| 404 | `not_found` | Group is not configured (should not occur in a correctly set-up installation) |
| 500 | `internal_error` | Unexpected server error |

## Examples

### curl

```bash
curl https://example.com/api/v1/locales
```

### fetch (TypeScript)

```typescript
type LocalesResponse = {
  default: string
  active: string[]
}

const res = await fetch('/api/v1/locales')

if (!res.ok) {
  const err = await res.json()
  throw new Error(err.message)
}

const locales: LocalesResponse = await res.json()
console.log(locales.default)  // "en"
console.log(locales.active)   // ["en", "es", "ca"]
```
