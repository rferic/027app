# GET /api/v1/locales

Gibt die aktiven Sprachen der Gruppe sowie die Standardsprache der Gruppe zurück.

Dieser Endpoint ist öffentlich — keine Authentifizierung erforderlich. Er ist für Clients gedacht, die die verfügbaren Sprachen kennen müssen, bevor ein Benutzer angemeldet ist (z. B. eine Login-Seite oder eine öffentliche Ansicht).

## Authentifizierung

**Keine** — dieser Endpoint ist öffentlich zugänglich.

## Anfrage

```
GET /api/v1/locales
```

Keine Header, Abfrageparameter oder Anfragekörper erforderlich.

## Antwort

**200 OK**

```typescript
type LocalesResponse = {
  default: string    // The group's default locale (e.g. "en")
  active: string[]   // All active locale codes, including the default
}
```

Sprachcodes folgen dem Format [IETF language tag](https://en.wikipedia.org/wiki/IETF_language_tag). Die Plattform unterstützt: `en`, `es`, `it`, `ca`, `fr`, `de`.

Beispiel:

```json
{
  "default": "en",
  "active": ["en", "es", "ca"]
}
```

## Fehler

| HTTP | `error` | Ursache |
|---|---|---|
| 404 | `not_found` | Gruppe ist nicht konfiguriert (sollte bei einer korrekten Installation nicht auftreten) |
| 500 | `internal_error` | Unerwarteter Serverfehler |

## Beispiele

### curl

```bash
curl https://example.com/api/v1/locales
```

### fetch (TypeScript)

```typescript
type LocalesResponse = {
  default: string
  active: string[]
}

const res = await fetch('/api/v1/locales')

if (!res.ok) {
  const err = await res.json()
  throw new Error(err.message)
}

const locales: LocalesResponse = await res.json()
console.log(locales.default)  // "en"
console.log(locales.active)   // ["en", "es", "ca"]
```
